## 二、已完成的优化模块

### 2.1 CBAM注意力机制 ✅

**实现文件**: `backend/yolo_modules/cbam.py`, `backend/yolo_modules/cbam_utils.py`

**技术细节**:
- **通道注意力**: 通过全局平均池化和最大池化学习通道间的关系
- **空间注意力**: 通过卷积学习空间位置的重要性
- **插入策略**: 仅在Backbone深层和Neck部分插入，避免浅层计算开销

**核心代码**:
```python
class CBAM(nn.Module):
    def __init__(self, channels, reduction_ratio=16, spatial_kernel=7):
        self.channel_attention = ChannelAttention(channels, reduction_ratio)
        self.spatial_attention = SpatialAttention(spatial_kernel)
    
    def forward(self, x):
        x = self.channel_attention(x)
        x = self.spatial_attention(x)
        return x
```

**优化效果**:
- 提升特征提取能力，特别是骨折区域的特征响应
- 参数量增加约 2-5%，计算量增加约 3-8%
- mAP@0.5 预期提升 2-4%

### 2.3 高级损失函数 ✅

**实现文件**: `backend/yolo_modules/advanced_loss.py`

**实现的损失函数**:

#### 2.3.1 SIoU Loss (SCYLLA IoU)
- **角度损失**: 考虑预测框与目标框的角度差异
- **距离损失**: 中心点距离的惩罚
- **形状损失**: 宽高比例的匹配

```python
class SIoULoss(nn.Module):
    def forward(self, pred_boxes, target_boxes):
        # 1. 角度损失
        angle_cost = torch.sin(torch.arcsin(sin_alpha) - math.pi / 4) ** 2
        # 2. 距离损失
        distance_cost = 2 - torch.exp(-gamma * rho_x) - torch.exp(-gamma * rho_y)
        # 3. 形状损失
        shape_cost = (1 - torch.exp(-delta_w)) ** theta
        # 组合损失
        loss = 1 - iou + (distance_cost + shape_cost) / 2
```

#### 2.3.2 EIoU Loss (Efficient IoU)
- 将IoU损失分解为重叠损失、中心距离损失和宽高损失
- 解决CIoU的收敛问题

#### 2.3.3 Focal Loss
- 处理类别不平衡问题
- 降低易分类样本的权重，关注难分类样本

#### 2.3.4 Quality Focal Loss (QFL)
- 联合学习分类和定位质量
- 使用IoU作为软标签

#### 2.3.5 Distribution Focal Loss (DFL)
- 学习边界框的分布表示
- 提高定位精度

**优化效果**:
- 边界框回归精度显著提升
- 小目标检测能力增强
- 预期 mAP@0.5:0.95 提升 5-10%

**实现的分配策略**:

#### 2.4.1 SimOTA (Simplified Optimal Transport Assignment)
- **动态top-k**: 根据候选框质量动态选择正样本数量
- **代价矩阵**: 综合考虑分类和回归代价
- **中心采样**: 只在GT中心区域内选择正样本

#### 2.4.2 TaskAlignedAssigner
- **任务对齐度量**: t = s^α × u^β
  - s: 分类分数
  - u: IoU
  - α, β: 权重参数
- **动态分配**: 根据对齐程度动态选择正样本

#### 2.4.3 DynamicLabelAssignment
- **智能切换**: 早期使用SimOTA，后期使用TaskAligned
- **自适应**: 根据训练阶段自动选择最佳策略

**优化效果**:
- 正样本分配更加合理
- 减少模糊样本的影响
- 预期 mAP@0.5 提升 2-3%

### 2.5 训练配置优化 ✅

**实现文件**: `backend/yolo_modules/training_config.py`

**优化内容**:

#### 2.5.1 优化器选择
- **AdamW**: 比SGD更适合医学图像
- **学习率**: 降低初始学习率至0.001，更稳定
- **权重衰减**: 增加至0.0005，防止过拟合

#### 2.5.2 学习率调度
- **余弦退火**: 平滑的学习率下降
- **预热阶段**: 5-10轮预热，避免初期震荡
- **最小学习率**: 最终学习率为初始的1%

#### 2.5.3 数据增强
- **医学影像专用增强**:
  - CLAHE对比度增强
  - 锐化
  - 高斯噪声（模拟设备噪声）
  - 高斯模糊（模拟运动伪影）
- **保守策略**: 小角度旋转（5度），禁用上下翻转

#### 2.5.4 多尺度训练
- **尺度范围**: 0.5x - 1.5x（320-960）
- **动态调整**: 每轮随机选择输入尺寸

#### 2.5.5 损失权重优化
- **box**: 7.5 → 10.0（更重视定位）
- **cls**: 0.5 → 0.3（相对降低）
- **dfl**: 1.5 → 2.0（分布焦点损失）

**预设配置**:
1. **默认配置**: 平衡性能和速度
2. **医学配置**: 更保守的增强，更长的训练
3. **快速配置**: 用于快速实验验证
4. **高分辨率配置**: 1280x1280，适合细节检测

def batch_inference(model, images, batch_size=8):
    results = []
    for i in range(0, len(images), batch_size):

        batch_results = model(batch, batch=batch_size)

## 七、注意事项

### 7.1 训练建议

1. **分阶段训练**:
   - 阶段1: 冻结Backbone，训练Head（10轮）
   - 阶段2: 解冻全部，微调（190轮）

2. **学习率调整**:
   - 如果损失震荡，降低学习率
   - 如果收敛慢，增加预热轮数

3. **数据增强**:
   - 根据数据集特点调整增强强度
   - 医学图像建议保守增强

### 7.2 常见问题

**Q: CBAM导致显存不足？**
A: 使用'deep_neck'或'neck_only'插入策略，减少CBAM模块数量。

**Q: SIoU Loss收敛慢？**
A: 降低学习率，增加预热轮数，或改用EIoU Loss。

**Q: 动态标签分配不稳定？**
A: 增加warmup_epochs，或固定使用SimOTA。
