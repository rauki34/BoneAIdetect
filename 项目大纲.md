# 医学骨折检测分析系统 - 项目大纲

---

## 一、项目介绍

### 1.1 项目背景
医学影像诊断是骨科疾病诊疗的重要环节，传统的人工阅片方式依赖医生经验，存在诊断效率低、主观性强等问题。本项目基于深度学习技术，构建了一套智能化的骨折检测分析系统，能够自动识别X光片中的骨折部位，并提供辅助诊断建议，为医生提供决策支持。

### 1.2 项目目标
- 实现骨折部位的自动检测与定位
- 提供多模型选择，适应不同检测场景
- 支持模型自定义训练与优化
- 集成AI辅助诊断，生成专业医疗建议
- 构建完整的检测历史管理与数据分析体系
- 保障用户账户安全的验证码机制
- 支持亮暗双主题切换，提升用户体验

### 1.3 核心功能模块
| 模块 | 功能描述 |
|------|----------|
| 图片检测 | 上传X光片，自动检测骨折部位，支持多模型选择 |
| 视频流检测 | 批量处理视频文件，逐帧检测，WebSocket实时推送 |
| 摄像头实时检测 | 连接摄像头，实时检测画面，即时显示结果 |
| 模型训练管理 | 上传数据集训练自定义模型，支持CBAM优化和模型续训 |
| 数据分析 | 检测统计、趋势分析、模型评估、用户个性化数据展示 |
| AI辅助诊断 | 基于多模态大模型生成医疗建议，支持多种AI服务提供商 |
| 用户认证 | 登录、注册（含验证码保护）、权限管理 |
| 系统监控 | CPU、内存、磁盘实时监控，系统统计信息 |
| 文件管理 | 上传、下载、删除文件，文件大小格式化显示 |
| 操作日志 | 完整操作记录审计，支持查询和筛选 |

## 六、技术难点与解决方案

### 6.1 大模型显存占用过高
**难度**: ⭐⭐⭐⭐

**问题**: Qwen3-VL-4B模型FP16精度需要约8GB显存

**解决方案**:
- 使用BitsAndBytes库进行4-bit量化
- 配置`load_in_4bit=True`、`bnb_4bit_quant_type="nf4"`
- 启用CPU offload，允许部分层在CPU执行
- 显存占用从~8GB降低到~2GB

**成果**: 成功在RTX 3060 6GB显卡上流畅运行大模型

### 6.2 模型训练CUDA内存错误
**难度**: ⭐⭐⭐

**问题**: 训练时出现 "CUDA error: resource already mapped"

**解决方案**:
- 设置 `workers=0` 禁用多进程数据加载
- 设置 `pin_memory=False` 禁用内存锁定
- 添加 `amp=False` 避免PyTorch 2.6兼容性问题

**成果**: 避免CUDA资源冲突，训练稳定运行

### 6.3 注册验证码实现
**难度**: ⭐⭐⭐

**问题**: 需要实现完整的验证码保护机制

**解决方案**:
- 使用Pillow生成验证码图片
- 添加噪点和干扰线防OCR
- Flask Session存储验证码
- 前端显示验证码图片并支持刷新
- 注册时验证码验证

**成果**: 完整的验证码保护机制

### 6.4 模型训练实时进度更新
**难度**: ⭐⭐⭐

**问题**: YOLO训练是同步阻塞的，无法实时获取训练进度

**解决方案**:
- 使用YOLO的回调函数`on_train_epoch_end`
- 每个epoch结束时更新数据库进度
- 前端定时轮询获取最新进度
- 支持训练任务手动终止

**成果**: 实现训练进度实时显示和任务控制

### 6.5 WebSocket实时视频检测
**难度**: ⭐⭐⭐⭐

**问题**: 视频文件处理时间长，用户无法感知处理进度

**解决方案**:
- 使用Flask-Sock建立WebSocket连接
- 后端异步处理视频，逐帧推送检测结果
- 前端实时显示处理进度和结果

**成果**: 实现视频检测的实时反馈

### 6.6 CBAM注意力机制集成
**难度**: ⭐⭐⭐⭐

**问题**: 需要动态修改YOLO模型结构插入CBAM模块

**解决方案**:
- 实现CBAM模块（通道注意力+空间注意力）
- 创建`create_yolo_with_cbam`函数动态修改模型
- 采用'deep_neck'插入策略，仅在Backbone深层和Neck插入
- 保存临时模型文件确保训练使用CBAM版本

**成果**: 成功集成CBAM，提升模型特征提取能力

### 6.7 亮暗主题切换
**难度**: ⭐⭐⭐

**问题**: 需要完整的主题切换适配

**解决方案**:
- 使用CSS变量定义主题颜色
- 通过`html.dark`类切换暗色模式
- 主题偏好持久化存储到localStorage
- 完整覆盖Element Plus组件样式

**成果**: 实现流畅的亮暗主题切换

- `training_tasks.model_id` → `custom_models.id` (外键关联)

本项目综合运用前后端分离架构、深度学习目标检测、大语言模型推理等多项技术，构建了一个完整的医学影像辅助诊断系统。通过4-bit量化、WebSocket实时通信、模型续训、CBAM注意力机制、验证码保护、亮暗双主题切换等技术手段，成功解决了大模型部署、实时检测、模型优化、账户安全、用户体验等关键问题，具备实际医疗场景应用的基础条件。

项目的模块化设计和完善的文档体系，为后续功能扩展和性能优化奠定了良好基础。
