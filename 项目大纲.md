# 智慧骨科云平台 - 项目大纲

## 一、项目概述

### 1.1 项目简介
智慧骨科云平台是一个基于深度学习的医学影像骨折智能检测与诊疗管理系统，采用前后端分离架构，支持管理员、医生、患者三种角色的协同工作。系统通过YOLO系列目标检测算法对X光片进行自动分析，并结合大语言模型提供专业的医学解读建议。

### 1.2 核心功能特性
- **三端分离架构**：患者端、医生端、管理员端独立设计
- **AI骨折检测**：支持图片、视频、摄像头三种检测方式
- **多模型支持**：YOLOv8、YOLOv11、YOLOv26及自定义训练模型
- **患者管理**：完整的患者档案、病历、检查记录管理
- **医生工作站**：患者管理、病历管理、AI辅助诊断一体化
- **医患关系**：支持主治医生与患者的绑定关系
- **医生入驻审核**：医生注册需管理员审核，确保资质合规
- **AI医疗建议**：基于检测结果生成专业诊断建议
- **消息通知**：医患之间即时通讯，系统公告定向推送
- **模型训练管理**：自定义模型训练、超参数优化、模型发布
- **数据统计分析**：检测趋势、患者统计、工作量统计
- **操作日志**：完整的操作审计记录

## 二、系统架构

### 2.1 整体架构
系统采用前后端分离的微服务架构，包含三个核心服务：

1. **前端服务**（Vue 3 + Vite）
   - 负责用户界面渲染和交互
   - 端口：5173（开发）/ 80（生产）

2. **后端API服务**（Flask）
   - 负责业务逻辑处理和数据管理
   - 端口：5000

3. **AI推理服务**（Flask + Transformers）
   - 负责大模型加载和医疗建议生成
   - 端口：8000

## 三、功能模块详细设计

### 3.1 统一登录入口

#### 功能描述
- 支持患者、医生、管理员三种角色登录
- 图形验证码保护，防止恶意注册
- 医生可申请入驻，需管理员审核
- 自动根据角色跳转对应首页

#### 用户角色
| 角色 | 权限范围 |
|------|----------|
| admin（管理员） | 系统管理、用户管理、模型管理、医生审核、公告管理 |
| doctor（医生） | 患者管理、病历管理、AI辅助诊断、消息通知 |
| patient（患者） | 查看病历、检查报告、主治医生、消息通知、AI咨询 |

#### 接口列表
- `POST /api/login` - 用户登录
- `POST /api/register` - 用户注册
- `POST /api/logout` - 用户登出
- `GET /api/captcha` - 获取验证码图片
- `POST /api/doctor/register` - 医生入驻申请

### 3.2 患者端功能模块

#### 功能描述
- **个人概览**：查看基本信息、主治医生、最新病历

- **主治医师**：查看主治医生信息
- **检查报告**：查看AI检测报告
- **消息通知**：接收医生消息和系统公告
- **AI咨询**：与AI助手对话咨询

#### 接口列表
- `GET /api/patient/doctors` - 获取主治医生列表
- `GET /api/patient/medical-records` - 获取病历记录
- `GET /api/patient/detection-reports` - 获取检查报告
- `POST /api/patient/ai/chat` - AI对话咨询
- `GET /api/patient/ai/history` - 对话历史

### 3.3 医生工作站模块

#### 功能描述
- **工作台首页**：统计数据、待办事项、快捷入口
- **患者管理**：我的患者列表、添加患者、患者详情、归档/恢复患者
- **病历管理**：创建病历、编辑病历、病历列表
- **AI辅助诊断**：图片检测、视频检测、摄像头检测
- **检查报告**：查看所有检测报告
- **检测历史**：历史检测记录查询
- **消息通知**：与患者沟通、接收系统消息

#### 接口列表
- `GET /api/doctor/dashboard` - 工作台数据
- `GET /api/doctor/patients` - 患者列表
- `POST /api/doctor/patients` - 添加患者
- `PUT /api/doctor/patients/{id}/archive` - 归档/恢复患者
- `GET /api/doctor/all-patients` - 所有患者（用于选择）
- `GET/POST /api/doctor/medical-records` - 病历列表/创建
- `PUT/DELETE /api/doctor/medical-records/{id}` - 病历更新/删除

### 3.4 管理员后台模块

#### 功能描述
- **仪表盘**：系统统计数据、趋势图表
- **用户管理**：创建/编辑/删除用户、重置密码
- **医生审核**：审核医生入驻申请
- **模型管理**：发布/禁用/删除模型
- **训练任务**：查看训练进度、停止任务
- **系统监控**：CPU、内存、磁盘监控
- **操作日志**：查看系统操作记录
- **公告管理**：发布公告、定向推送（医生/患者/全部）

#### 接口列表
- `GET /api/admin/dashboard` - 仪表盘数据
- `GET /api/admin/statistics` - 系统统计
- `GET /api/admin/doctor-registrations` - 医生注册审核列表
- `POST /api/admin/doctor-registrations/{id}/review` - 审核医生注册
- `GET/POST /api/admin/announcements` - 公告列表/创建
- `PUT/DELETE /api/admin/announcements/{id}` - 公告更新/删除

### 3.5 AI检测模块

#### 功能描述
- **图片检测**：上传单张或多张图片检测
- **视频检测**：上传视频文件逐帧检测
- **摄像头检测**：实时摄像头检测
- **多模型切换**：支持YOLOv8/11/26及自定义模型
- **患者关联**：检测结果可关联到患者档案
- **AI解读**：基于检测结果生成医疗建议

#### 检测流程
```
用户上传图片 → 图片预处理 → 模型推理 → 结果标注 → 生成报告 → 关联患者
     │              │           │          │          │           │
     ▼              ▼           ▼          ▼          ▼           ▼
  格式验证    尺寸调整归一化  目标检测   绘制检测框   保存记录    选择患者
```

#### 接口列表
- `POST /api/predict` - 单张图片检测
- `POST /api/video/detect` - 视频文件检测
- `POST /api/camera/detect` - 摄像头实时检测
- `GET /api/history` - 检测历史
- `GET /api/history/{id}` - 检测详情
- `POST /api/history/{id}/advice` - 保存医疗建议
- `POST /api/interpret` - AI解读检测

### 3.6 模型训练管理模块

#### 功能描述
- **数据集管理**：上传YOLO格式数据集
- **训练配置**：设置轮数、批次大小、图像尺寸
- **超参数优化**：基于Optuna自动优化超参数
- **训练监控**：实时查看训练进度和损失曲线
- **模型评估**：自动计算mAP、Precision、Recall、F1
- **模型发布**：训练完成后发布供使用

#### 训练参数
- **优化器**：AdamW、SGD、RMSprop
- **学习率**：0.0001 - 0.01
- **批次大小**：8、16、24
- **数据增强**：mixup、degrees、scale等
- **早停机制**：patience参数控制

#### 接口列表
- `GET /api/models` - 获取模型列表
- `GET /api/models/published` - 获取已发布模型
- `POST /api/models/train` - 创建训练任务
- `POST /api/models/{id}/publish` - 发布模型
- `POST /api/models/{id}/disable` - 禁用模型
- `POST /api/models/{id}/enable` - 启用模型
- `DELETE /api/models/{id}` - 删除模型
- `GET /api/training/tasks` - 训练任务列表
- `GET /api/training/tasks/{id}/progress` - 训练进度
- `POST /api/training/tasks/{id}/stop` - 停止训练

### 3.7 消息通知模块

#### 功能描述
- **消息联系人**：获取可发送消息的联系人列表
- **发送消息**：医患之间即时通讯
- **未读消息**：实时显示未读消息数量
- **系统公告**：管理员发布公告，定向推送给医生或患者

#### 接口列表
- `GET /api/messages/contacts` - 消息联系人
- `GET /api/messages/conversation/{user_id}` - 聊天记录
- `POST /api/messages/send` - 发送消息
- `POST /api/messages/mark-read` - 标记已读
- `GET /api/messages/unread-count` - 未读消息数
- `GET /api/announcements` - 公告列表
- `POST /api/announcements/{id}/read` - 标记公告已读

### 3.8 数据统计分析模块

#### 功能描述
- **检测趋势**：按日/周/月统计检测数量
- **置信度趋势**：检测置信度变化曲线
- **患者统计**：年龄段分布、性别分布
- **工作量统计**：医生工作量统计
- **模型使用统计**：各模型使用频率

#### 接口列表
- `GET /api/analysis` - 全局统计分析
- `GET /api/analysis/confidence_series` - 置信度趋势
- `GET /api/analysis/user_confidence_series` - 个人置信度趋势
- `GET /api/analysis/user_stats` - 个人统计
- `GET /api/monitor/system` - 系统监控
- `GET /api/monitor/stats` - 监控统计

        proxy_set_header X-Real-IP $remote_addr;
    }

    # AI服务代理
    location /ai {
        proxy_pass http://127.0.0.1:8000;

    }
